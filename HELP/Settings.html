Overview

The settings are displayed in the following four tabs :

  • Software , where software and git settings are handled
  • Repository , where repositories are handled
  • Branch , where branches of repository selected in "Repository tab" are handled
  • System info , which display information relevant to support, and the system path as seen by Pragma-git

Repository tab

The repository settings have two parts :

  • Repository table , where you may select one of the repositories you have defined
  • Tools , a set of tabs with tools acting on repositories

The "Repository table"

The rows shows the settings for the repositories you have defined so far.

The first column "Local folder" shows the path to your active repositories, on your local pc. The radio-button, can be used to select one row, and the second table below shows the branches in that selected repository

The second column "Remote repository" shows the address to a connected remote repository. If this is not defined, then you work with a local repository ( which is fine as long as you have a backup plan)

The third column has the button, which is used to test that the remote repository can be connected to, and at the same time storing the address

The forth column has the button, which is used remove the selected row from the list (it can always be added again, just as you did the first time by dropping the folder on the main window)

The "Tools tabs"

Different tools can be used to work on repositories. They are selected from the tool tabs.

This tab handles settings that are specific for the currently selected repository.

Author's information


The author information here may override the global author information for this selected repository (see "Software" tab)

Author's name textfield — Required field. Modify your name (on all your repositories on this computer)

Author's email textfield — Modify your email (on all your repositories on this computer)

Use global (from Software tab) —
When this is unchecked, the author's name and email fields are used instead of the global values. If checked, global values are used and displayed.

Git behavior


Auto-push to remote, when Store clicked (git commit; git push). Deselect to only commit (git commit) —
Pragma-git has the possibility to auto-push if a remote repository exists. This is handy most times, but it can be turned off by deselecting this checkbox.

Merge without fast-forward (git merge --no-ff). History is not rewritten. Deselect to use fast-forward —
Fast-forward merge is a git feature that is debated. Some developers love it because it may make the commit history appear cleaner than it really was, and some thinks it is very awkward since it changes the history. It is recommended that beginners have this checkbox selected, so that the history follows what has actually happened.


This tab shows the .gitignore file, which is used to stop new files (matching a pattern) from being added to the repository. It can be viewed as a filter against files that change by theirselves when you develop. This could for instance be compiled files, log files, or automatically generated backup files

opens a special editor for .gitignore files, that aids in creating and modifying the filter conditions


Instead of starting with a local folder, you may connect with an existing repository (this process is called cloning).

Cloning a remote repository :

  1. The first text field is used to define the remote repository address (url)
  2. button is used to verify that you can connect to the defined remote repository
  3. button is used to select the folder, into which the cloned repository is copied.
  4. The second text field displays the selected folder path
  5. makes a copy of the remote repository inside the selected folder, and connects your local copy to the remote repository

A local folder can be added using the add tool. If the local folder is not a repository, a dialog will ask if you want to create a git repository from the selected folder. If the folder is a repository already, its settings are imported automatically.

Adding a local folder :

  1. button is used to select the folder, into which the cloned repository is copied.
  2. The text field displays the selected folder path
  3. adds the repository inside the selected folder, and asks to make the folder a repository if necessary.

This is the same functionality as dropping a folder to the Pragma-git main window.


Adding a remote repository :

For remote git services, enter remote URL manually for selected repository.
Format : https://name:password@address (for token-based services, exchange name for token-name, and password for token)

The password, and token, may be left out, and instead stored with the git credential manager.
Format : https://name@address . If you have problems, fall back to method above.

The Github tool aids in setting up the URL of the remote repository on Github. It interacts with your Github account, allowing for creation of access tokens, or to create a new repository.

button launches a Github dialog. It has a separate help function


Some words about remote repositories

It is already described, above, how you link your project to a remote repository; and also how you clone a remote repository to your local computer. This section describes how to set up user-names and passwords for remote repositories.

(I) Creating an account

There are multiple remote repository hosting services that you can use on internet. One popular site is Github (by Microsoft), which gives free accounts.

Get an account, go to the web interface, and create a new repository. You will get a link on the format https://github.com/JanAxelssonTest/test.git
which you need to copy.

(II) Authentication

Login to a repository may require some kind of authentication. Some alternatives are :

  1. Access token (Github has moved towards this -- see link
  2. Old fashioned user-name and password (this is being phased out by many git services, but may be applicable if you run your own git repository)

(III) Using your link (address / url)

Paste the link to your repository into the text-field under "Remote repository".

Alternative 1) access tokens, modify the link by adding the ACCESSTOKEN, and an @-sign :
https://ACCESSTOKEN@github.com/USERNAME/REPOSITORY_NAME.git. For instance :
https://25badab677c62a7a3e44e5dd123a916adab8580677e0ffbd@github.com/JanAxelssonTest/test.git

Alternative 2) username and passwords, modify the link by adding user:password, and an @-sign :
https://USER:PASSWORD@github.com/USERNAME/REPOSITORY_NAME.git For instance :
https://JanAxelssonTest:PASSWORD@github.com/JanAxelssonTest/test.git

The repository is set (and tested) by pressing the button. If the text becomes green, the remote repository is correctly defined. Adjust your remote repository text until the "Set" button creates a green text, then it should work.

If git is using a credential manager, you can now remove the :password (note the :code>) and test if it now works without storing the password in the link. Also, if git forgets the password after a while, you will need to keep the password field or install a credential manager. Mac and Windows should be fine by default.

Branch tab

The branches for a selected repository (in the repository table) are listed in all but the last row.

Hide checkbox, allows a branch to be hidden from branch selection in the main window. The point of that is to reduce clutter, if you have a lot of branches but don't want to delete them.

A branch may be deleted, obviously something you only do if you are done working on that branch. This could mean either that 1) you have merged your work to another branch, or 2) that your work led to a dead-end and you wish to discard it. Press the button, and confirm in the dialog poping up if you wish to delete this branch, or cancel.

In the last row, you can add a new branch, by typing its name, and pressing the button. The branch name cannot contain all characters, so the text field stops you from typing forbidden characters

Software settings tab

The software settings are grouped as follow :

Visual

- Dark mode setting , changes the visual theme (requires restart of Pragma-git). Options are:
System setting — follow the system-defined light/dark theme
Light mode — light theme
Dark mode — dark theme
- Window behavior
Always on top — makes the main window always be on top of other program windows
Show on all workspaces — shows the Pragma-git on all virtual workspaces (this is a Mac and Linus thing)
Show tooltips in main window -- shows tooltip when hovering over most buttons (not history arrows or Store button)

- Windows scaling
Scale-factor for main window — changes main-window scaling (useful for high definition screens)
Scale-factor for other windows — for instance a setting of 0.75 means the size is 75% of default size

Git Settings


These settings are common for all repositories. The author information may however be overridden by the setting in an individual repository (see "Repository / Software (this repo)" tab)

Author's name textfield — Required field. Modify your name (on all your repositories on this computer)

Author's email textfield — Modify your email (on all your repositories on this computer)

Branch change only when all files are committed — Stops you from changing branch, if you have not Stored your data. This is the safe way to work.

Search and history follow current branch — This is default, and makes history follow the current branch. Similarly, search look in the current branch. This makes searches less confusing, since the result follow a logical flow, and does not jump back and forth between branches. However: Results where fast-forward merge (above setting) are used will be confusing, and if fast-forward-merges are your settings, it is recommended to disable the "Search in current branch" setting.

Click both applies and removes stash (git stash pop). Deselect to apply and keep stash ( git stash apply) — Background information: Stashed files can be popped back into the repository. When a stash is applied, there are two options : 1) remove the stash, or 2) keep the stash.

Git tools

Pragma-git has a built-in diff and merge tool. If the fields for external tools are left blank (or says pragma-git), the built-in tool will be used.

If you have your favorite Diff and Merge tool it may be used instead.
There are numerous such tools to chose from (search internet for : Diff and Merge tool). Some that are listed in the git help are : meld, kdiff3, kompare, tkdiff, xxdiff, emerge, gvimdiff, ecmerge, diffuse, opendiff, p4merge and araxis. Meld exists for Windows, Mac, and Linux, but there are many platform specific tools that may be equally good, or even suit you better. The settings for external tools are :

- Diff tool , lets Pragma-git show you differences between different revisions. The diff-program must be installed separately. One good diff program is "Meld", but there are tens of different programs to download

- Merge tool , lets Pragma-git help you solve file conflicts where git cannot decide safely on its own (conflicts arising from Merges, or Pull from remote repository where work has been done in both places)

- Addition to path , if the external diff/merge tools do not open, their path may not be accessible to Pragma-git. This can be fixed, by adding the path to the tools in this field. If you have different diff and merge tools, two paths may be needed. The two paths should then be separated by ";" for Windows, or ":" for Mac and Linux.
