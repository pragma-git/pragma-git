You are in this window because there are file conflicts, most likely caused by a Merge or Stash-pop operation. The reason is that a file has been deleted or modified in one place, or that there are differences in overlapping parts of a file. These cases needs a manual intervention, to decide which change to keep.

If you want to undo what caused the conflict

Press in case you wish to undo the Merge or Stash-pop operation
(or the Stash pop, which has caused the conflict because a merge is part of that operation)


If you want to solve the conflict

Each type of conflict is solved in a different section

Resolve the conflicts starting at the top section. You may fold the sections, by clicking on the header.

Section "1) Select files to keep"

The files listed here were in a state where it could not be determined if they should be kept, or deleted. Check files you wish to keep, and click the button :

Section "2) Conflicting files"

Clicking the button starts a merge tool (see Requirements, below). This button will be enabled when there are no conflicts in step 1).

Section "3) Accept changes"

This is your last chance to change your mind.

  • Press in case you are not sure the conflict was resolved correctly
  • Press to finish. (This will also remove some backup-files created when working with the conflicts)
NOTE : After this, you should type a commit description and press the Store-button in the main window

Requirements

This window uses either the built in merge tool, or an external merge tool that is defined in the settings window.
