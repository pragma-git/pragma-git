You are in this window because there are file conflicts. This could be because the file has been deleted or modified in one place, or that there are differences in overlapping parts of two files. Both these cases needs a manual choice, which change to keep.

Each type of conflict is solved in a different section

Resolve the conflicts starting at the top section. You may fold the sections, by clicking on the header.

Section "1) Select files to keep"

The files listed here were in a state where it could not be determined if they should be kept, or deleted. Check files you wish to keep.

Section "2) Conflicting files"

Clicking the "Solve conflicting files"-button starts a merge tool (see Requirements, below). This button will be enabled when there are no conflicts in step 1).

Section "3) Accept changes"

This is your last chance to change your mind.

  • Press "Undo Merge" in case you are not sure the conflict was resolved correctly
  • Press "Click to Accept" to finish. (This will also remove some backup-files created when working with the conflicts)
NOTE : After this, you should type a commit description and press the Store-button in the main window

Requirements

This window uses either the built in merge tool, or an external merge tool that is defined in the settings window.
