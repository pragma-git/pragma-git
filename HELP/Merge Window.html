Introduction

This window shows differences between two commits. It can also be used as a difference editor, where uncommited files can be compared to a previous state, and each change can be edited.

This window is also activated when conflicts between file versions are found in merges between two branches, or between the remote and the local repository.

Frames -- diff viewer/editor

In difference mode, 2 frames are shown. They are titled according to their function (new / stored / selected / previous). Generally speaking, the left frame displays the modified file and the right the already stored file.

Frames -- merge editor

In merge mode, you can select to see 2 or 3 frames ( or button). 2 frames works as in difference mode. In 3 frame mode, the middle frame is the result from your merge, and changes can be selected to be moved into the middle frame from either of the two files displayed in left and right frames.

Editing

When editing or merging of two versions is allowed, small arrows can be clicked to copy the adjacent rows.

Save

Message "Note: executable bit differs"

This message indicates that the executable bit (chmod +x command on linux and MacOS) is different between the files.

When executable bit differs between files, the title above the code panes (showing : new / stored / selected / previous) is amended showing the executable bit status.
(x+) means that the file is executable.
(x-) that it is non-executable.
