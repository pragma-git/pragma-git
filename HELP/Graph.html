Overview

The graph window consists of multiple parts :

  • Header (top) , shows repo and branch name
  • Graph view (left) , displays the git-log graph
  • Branch list (right) , gives color coding for nodes, and some display setting checkboxes

Header (top)

Current repository and branch

The current repository is displayed in the header, together with the branch and a circle showing its display color (within parenthesis). The header for repository "pragma-git" and branch "develop" could look like this : pragma-git ( develop)

Mode switch

The mode switch " compress lanes " determines how branches with unknown names are displayed

Branches with known names can be displayed two ways :

  • compress , where graph takes minimal space. A branch does not follow a fixed column.
  • lanes , (sometimes called swim-lanes) where each known branch is placed in its own horizontal position

Graph view (left)

Commits

Commits are shown as circles.

Colors

The graph shows commits of known branch-names as color nodes, for instance : ...

For commits with unknown branches, the graph displays commits on the current branch in green , and commits on other branches without color .

Due to how git works, the current branch can only be deduced correctly if the setting "Merge without fast-forward" is used (recommended).

Commit texts

Commit message texts outside the current branch are displayed in italics and a diffent color. Also, commits not found in the Search box (of the main window) are displayed in italics and a diffent color.

Zoom

Holding ctrl-key while using the mouse-wheel zooms the size of the graph

Navigating

Moving the mouse pointer over the commit message, a grey shaddow appears, together with a checkbox .

Select commit

Clicking on the commit makes the text green, and moves the main window history to the selected commit. The main window can be used as before, looking for changes, clicking on the modified files to get a list of changes to prior commit, etc.

Checkbox, compare with fixed commit

Clicking the checkbox , makes the checked commit a fixed historical commit ().

Selecting the text of any other commit (not the checkbox) moves history to the newly selected commit (both in graph and in main window). The bottom of the main window is updated to display number of modified files between the selected and the fixed (checkboxed) commits.

Stashes / Work in progress

Stashed files ( from main window) can be retrieved by clicking the icon. The stashes are numbered, with the lowest number being newest. The icon is removed when a stash is retrieved, except if it causes a file conflict.

Commit Info

Pointing the mouse to a commit circle displays information about the commit, and the parents of that commit. Also, the circles for the commit and its nearest parent are enlarged, giving a visual view of their relationship.

Branch list (right)

This box displays the branch names and their color codes, together with some settings on the graph display.

The following settings modify the graph display :

Dates

, enables or disables display of commit date (git's "author's date") .

One date is shown per day, and newer commits are higher up. Dates are shown in ISO format, with YYYY-MM-DD (YYYY = four digit year, MM = two-digit month, DD = two-digit day)

All branches

, enables or disables display of branches that are not direct parents to the current branch. Showing all branches is messy in complex projects, and having this option turned off works most of the time.

Color hidden branches

, this option is only visible if any branches are hidden (*).

If checked, hidden branches are shown with branch colors ( ...) , otherwise the commits on hidden branches are shown in same color as unknown branches ( ).

(*) Branches can be hidden using the settings icon ( click main-window icon, and then Branch tab).
