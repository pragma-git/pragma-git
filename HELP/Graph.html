Current branch

The commit tree shows commits on the current branch in green , and commits on other branches without color . Selection in the graph window are synchronized with the main window.

Due to how git works, the current branch can only be deduced correctly if the settings "Merge without fast-forward" is used (recommended).

Text color

Commit message texts outside the current branch are displayed in italics and a diffent color. Also, commits not found in the Search box (of the main window) are displayed in italics and a diffent color.

Navigating

Moving the mouse pointer over the commit message, a grey shaddow appears, together with a checkbox .

Select commit

Clicking on the commit makes the text green, and moves history to the selected commit. The main window is updated to display this commit. The main window can be used as before, looking for changes, clicking on the modified files to get a list of changes to prior commit, etc.

Compare with fixed commit

Clicking the checkbox , makes the checked commit a fixed historical commit ().

Selecting the text of any other commit (not the checkbox) moves history to the newly selected commit (both in graph and in main window). The bottom of the main window is updated to display number of modified files between the selected and the fixed (checkboxed) commits.

Dates

, enables or disables display of commit date (git's "author's date") .

One date is shown per day, and newer commits are higher up. Dates are shown in ISO format, with YYYY-MM-DD (YYYY = four digit year, MM = two-digit month, DD = two-digit day)
