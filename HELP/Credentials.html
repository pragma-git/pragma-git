Overview

The credential dialog pops up when login details are missing for the remote git server. A credential helper should come with your git installation (Windows, MacOS), and will store the entered secret password or access token for you. Thus, you should only need to enter the credentials once.

Different git servers store logins in different ways. Generally, some sort of name (user name, account name, etc), is followed by some sort of secret (access token, or password).

Login methods

Below are different types of login methods, for remote git repositories :

1) Access Tokens

Typically, an access token is entered with a name and the access token

  • User name / token
  • Account name / token

2) Username and password

This is the old mode, which is more or less out of fashion with cloud servers such as Github :

  • User name / Password

Store in URL

The user name can be stored in the git URL, and can be entered in the repository section of the settings :

a) Store the user name (you will be asked for credentials :

https://USERNAME@github.com/USERNAME/REPONAME.git

b) Store the user name and access token (the Token will be readibl in the settings)

https://USERNAME:TOKEN@github.com/USERNAME/REPONAME.git https://TOKENNAME:TOKEN@gitlab.com/USERNAME/REPONAME.git

How to stop dialogs

The credential dialogs pops up when you do not have correct access rights. If this keeps happening, it can be fixed by changing the settings for the current repository to give correct access, by one of the following methods :
  • Enter the correct credentials (this works if you have a credential helper, which may not work on some systems
  • Disable the "Allow git push" setting if you don't have writing rights to remote repository
  • Add correct credentials in the URL
  • Remove the remote repository, in case you don't have reading / writing rights
