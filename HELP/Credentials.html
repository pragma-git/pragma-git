Overview

The credential dialog pops up when login details are missing for the remote git server. A credential helper should come with your git installation, and will store the entered secret password or access token for you. Thus, you should only need to enter the credentials once.

Different git servers store logins in different ways. Generally, some sort of name (user name, token name, etc), is followed by some sort of secret (password, or access token).

Login methods

Below are different types of login methods, for remote git repositories :

1) General Access Tokens

Typically, an access token is entered with a name and the access token

  • User name / token name
  • Access token

2) Username and password

This is the traditional mode, which is more or less out of fashion with cloud servers such as Github :

  • User name
  • Password

Store in URL

The user name can be stored in the git URL :

a) Store the user name

https://USERNAME@github.com/USERNAME/REPONAME.git

b) Store the user name and access token (not recommended, because the Token will be readible)

https://USERNAME:TOKEN@github.com/USERNAME/REPONAME.git https://TOKENNAME:TOKEN@gitlab.com/USERNAME/REPONAME.git
